# ORIGAMI 次に行う作業手順

この文書は、現在の状態から正式完成までに行う作業を、実行する順番でまとめたものです。

## 現在の状態

- Web画面、CLI、候補検証、失敗時の安全な停止処理は実装済み。
- 単体テスト、lint、型確認、ビルドは成功済み。
- ただし、正式な基準データと実測結果が未承認のため、最終完成判定はまだ行わない。

## 1. 正式な基準データを決める

### 作業

次の5種類を正式な基準データにする。

1. うさぎの耳
2. 水風船
3. 4枚構成の鳥
4. 長方形の箱ひだ
5. 平らに折りたためる簡単な形

各データについて、次を1つの資料にまとめる。

- 入力する紙の大きさ
- 面の数
- 折り線の一覧
- 折る順番
- 折り終わったときの面の位置
- 正しい結果と、間違いとする条件

### 確認

- 同じ入力から毎回同じ結果になる。
- 入力、期待結果、作成者、作成日が記録されている。
- 変更した場合は、変更理由と新しい確認結果を残す。

### 完了条件

5種類すべてに、入力と期待結果がそろい、別の人が見ても再現できる。

## 2. 最初に対応する範囲を固定する

最初の正式版では、対応範囲を次に限定する。

- 面の数は2〜20枚。
- 紙は正方形。
- 折り線は直線。
- 折りは1本ずつ順番に進む。
- 範囲外の入力は、理由を表示して処理しない。

### 確認

- 範囲内の入力が受け付けられる。
- 面が1枚、または21枚以上の入力が拒否される。
- 正方形でない紙や、対応外の折り方が拒否される。

### 完了条件

対応範囲が画面、CLI、説明書で一致している。

## 3. 実際のパソコンで処理限界を測る

### 作業

Windows上で、Chrome、Edge、Firefoxを使って測定する。

- 入力を読み込む時間
- 検証が終わるまでの時間
- 面数を増やしたときの変化
- 処理をキャンセルしてから止まるまでの時間
- 大きな入力で画面が固まらないか

各測定は最低3回行い、元の入力と結果を保存する。

### 確認

- 測定したパソコン、OS、ブラウザの版を記録する。
- 測定途中で失敗した場合も削除せず保存する。
- 最も遅い環境でも画面が操作不能にならない。

### 完了条件

安全に処理できる面数、処理時間、入力サイズの上限を決め、画面と説明書に反映する。

## 4. 折り途中の安全確認を完成させる

折り途中の各段階で、次を確認する。

- 面同士が突き抜けていない。
- 面のつながりが途中で切れていない。
- 同じ面が同時に別の場所へ存在しない。
- 折り重なりの上下関係が矛盾していない。
- 折り線の端点が前後の段階で一致している。

判断できない場合は、成功扱いにせず「確認できない」として停止する。

### 完了条件

正常例、突き抜け例、つながり切れ例、上下関係の矛盾例をそれぞれ用意し、期待どおりの結果になる。

## 5. 解がない場合の説明を実装する

解が見つからない場合は、次を保存して表示する。

- 調べた範囲
- 調べた候補数
- 拒否した理由
- 調査結果の識別用ハッシュ値

すべて調べ切っていない場合は「解なし」と断定せず、「範囲内では確認できなかった」と表示する。

### 完了条件

同じ入力を再実行すると同じ調査結果になり、途中で改ざんされた結果を検出できる。

## 6. Web画面から最後まで操作する

次の利用手順を実際に確認する。

1. `npm run dev`を実行する。
2. `http://127.0.0.1:5173/`を開く。
3. 入力データを選ぶ。
4. 検証を開始する。
5. 進捗を確認する。
6. 成功、失敗、未確認の違いを確認する。
7. 必要な結果を保存する。

### 完了条件

初めて使う人が説明を読めば操作でき、失敗時に次に何をすべきか分かる。

## 7. 最終判定を記録する

次の条件をすべて満たしてから、最終判定を記録する。

- 5つの正式基準データに合格している。
- 対応範囲と上限が決まっている。
- 折り途中の突き抜けとつながり切れを検出できる。
- 解がない場合の説明を再現できる。
- Chrome、Edge、Firefoxで安定して動く。
- ライセンスと出典を確認している。
- すべての証拠を別の環境で再確認できる。

条件が1つでも未達なら、最終判定は「未完了」のままにする。

## 実行時の基本ルール

- 仕様が決まっていない値を推測して正式仕様にしない。
- 確認できないものを成功扱いにしない。
- 変更ごとにテスト、lint、型確認、ビルドを実行する。
- 検証が成功した変更だけをcommitしてpushする。
- 完成度は、未完了の条件を含めて全体の割合で報告する。
