## 1. 確定事項サマリ

| 項目 | 決定 |
|---|---|
| プロダクト | 折り紙設計Webアプリ(展開図自動生成+3D折り畳みシミュレーション) |
| 想定ユーザー | 創作折り紙の上級者(展開図が読める層) |
| 設計方式 | ツリー法・箱ひだの両対応、同時並行開発(共通コア方式) |
| 折り方出力 | 山谷付き展開図+3Dシミュレーション(折り順図解は非対応) |
| 膨らみ | (a)カド幅=Phase 1後半 / (b)段折り立体化=Phase 2 / (c)3D曲面=Phase 3(スコープ外扱い) |
| 用紙 | 正方形・任意比率の長方形 |
| 出力 | FOLD / SVG / PDF / PNG |
| 保存 | ローカルのみ(プロジェクトJSONのDL/読込)。バックエンドなし |
| 配布 | 完全静的Webサイト(GitHub Pages)。将来Tauriデスクトップ化可 |
| ライセンス | MIT(依存もMIT互換に限定) |
| UI言語 | 日本語のみ(文字列は1モジュールに集約し将来のi18nに備える) |

## 2. 機能要件(実装レベル)

### FR-1 プロジェクト管理
- FR-1.1 新規作成(用紙形状・方式を選択)
- FR-1.2 プロジェクトファイル(.orid.json)の保存(ダウンロード)・読込(ファイル選択/ドラッグ&ドロップ)
- FR-1.3 Undo/Redo(全設計操作対象、履歴100件)
- FR-1.4 ブラウザリロード時の作業復元(IndexedDBへの自動スナップショット)

### FR-2 ツリーエディタ(設計入力)
- FR-2.1 キャンバス上でノード追加・ドラッグ移動・削除、枝(エッジ)の接続
- FR-2.2 各枝の属性: 長さ(正の実数、単位なし比率)、ラベル、幅w(膨らみa、デフォルト0)
- FR-2.3 数値テーブル: 全枝を一覧表示し長さ・幅・ラベルをインライン編集。キャンバスと双方向同期
- FR-2.4 左右対称モード: 縦軸対称を指定すると軸の片側の編集がもう片側へミラー反映
- FR-2.5 バリデーション: ツリーが連結・閉路なし・葉が2以上であることを常時チェックし違反を表示

### FR-3 ソルバー(展開図生成)
- FR-3.1 共通: パス条件(全葉ペアの紙上距離≥スケール×木距離)の検証と違反ペアのハイライト
- FR-3.2 箱ひだ: 格子数Nの指定と自動提案。葉ノードの格子上自動配置(ヒューリスティック)+手動ドラッグ調整モード
- FR-3.3 ツリー法: 連続最適化による自動配置(スケール最大化)+手動微調整モード
- FR-3.4 分子充填(共通): アクティブポリゴンへのuniversal molecule構築で折り線を生成
- FR-3.5 山谷割り当て: 自動割り当て(ヒューリスティック)+CPビュー上での手動M/V編集
- FR-3.6 検証: 前川定理・川崎定理の内部頂点チェック、違反頂点を赤表示。エラー時は原因メッセージを表示
- FR-3.7 解なし時: 違反しているパス条件のペアを指摘するエラーフィードバック

### FR-4 展開図ビュー・出力
- FR-4.1 CPの表示(山=赤実線/谷=青破線/境界=黒。慣例準拠、色は設定可)
- FR-4.2 ズーム・パン、格子表示(箱ひだ時)、参照点座標の表示
- FR-4.3 エクスポート: FOLD(v1.1準拠) / SVG / PNG(解像度指定) / PDF(用紙サイズ・余白指定)

### FR-5 3Dシミュレーション
- FR-5.1 生成CP(またはインポートFOLD)を三角形分割し、折り畳み率スライダー0–100%で3D表示
- FR-5.2 ビュー操作(回転・ズーム・パン)、面の表裏色分け
- FR-5.3 「全折り線同時進行であり折り順ではない」旨の注記を常時表示
- FR-5.4 シミュレーション破綻(自己交差等)時の警告表示

### FR-6 膨らみ
- FR-6.1 (a)幅: 枝の幅w>0のとき、葉円半径をr+w/2に拡張(ツリー法)/格子帯をw分拡幅(箱ひだ)し、CPに肉領域を確保
- FR-6.2 (b)段折り: 枝を選択し段数n・深さdを指定→当該カドの軸方向に段折り線対を挿入。消費長の警告(Phase 2)

## 3. 非機能要件

- NFR-1 性能: 葉20枚規模のツリーで、箱ひだ自動配置≤3秒、ツリー法最適化≤30秒(Web Worker実行、進捗表示、中断可)
- NFR-2 対応環境: デスクトップ最新版Chrome/Edge/Firefox/Safari。画面幅1280px以上を主対象
- NFR-3 静的配信: ビルド成果物のみで動作(外部API・サーバー通信なし)
- NFR-4 品質: ソルバー数学部のユニットテストカバレッジ80%以上、定番例題のゴールデンテスト
- NFR-5 ライセンス順守: 依存はMIT/BSD/Apache-2.0系のみ。GPL系依存は導入禁止

## 4. スコープ外(明記)

- 折り順のステップバイステップ図解の自動生成
- 3D曲面設計(Origamizer系)— Phase 3として将来検討、本計画には含めない
- クラウド保存・アカウント・作品共有機能
- モバイル最適化(表示可能であればよい)
