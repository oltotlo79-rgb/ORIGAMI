# OriDesign 受入テスト仕様書 v1.1

作成日: 2026-07-14  
ステータス: 実装準備版（M0F で確定する値は確定後に置換）

## 1. 目的

本書は「機能が動く」だけでなく、検証済み CP の保証境界、データ保全、相互運用性、無料静的配布を客観的に判定する。各マイルストーン AC との対応は`07_要件トレーサビリティ.md`で管理し、直接記載したtest IDと合わせて参照する。

## 2. 合否原則

- 自動化可能な AC は CI に置く。目視だけで合格にしない。
- 数理 fixture は入力、期待する分類、許容誤差 profile、期待する不変条件を version 管理する。
- 乱数を使うテストは seed を固定し、失敗時に seed と入力を保存する。
- 座標配列の単純 snapshot ではなく、canonical ordering と幾何同値比較を使う。
- 「検証済み」は `VerificationRecord` と証跡の再検証が通る場合だけ期待値にできる。
- 対応範囲内の入力に、タイムアウト成功／失敗という期待値を設けない。CI では watchdog を異常検知として使い、ドメイン結果の `no-solution-certified` と区別する。
- flaky test の再実行だけで CI を通さない。原因を特定するまで release gate を閉じる。

## 3. テスト層

| 層 | 対象 | 主な手段 |
|---|---|---|
| M0F experiment | 構成法、経路、CCD、停止性、対応範囲 | 固定 fixture、adversarial input、計測 |
| unit | model、geometry、solver、verify、migration | Vitest |
| property | 幾何不変条件、tree、planar graph、round-trip | 固定 seed の生成テスト |
| golden | CP、FOLD、検証証跡の意味的期待値 | canonical FOLD／certificate |
| integration | worker、state、persistence、export | Vitest browser または Playwright |
| E2E | 主要ユーザーフロー、復旧、PWA | Playwright |
| compatibility | 外部 FOLD、3 browser、PDF/SVG/PNG | fixture + 手動補完 |
| performance | solver target、3D fps、memory | 固定基準端末 + CI trend |
| accessibility | keyboard、name/role/value、contrast | axe 等 + keyboard E2E |

非canonical `NEG-SUPPORT-CATALOG-*`候補は未選択candidate catalogのclaim-boundary回帰に限定する。上表のcanonical `NEG-SUPPORT-BOUNDARY-*`、frozen SupportProfile、実入力の対応可否、termination保証を満たすfixtureとは数えない。

## 5. 数理・幾何 AC

### GEO-001 robust predicates

- orientation、segment intersection、point-in-polygon が平行、共線、近接、座標尺度違いの fixture を正しく分類する。
- ToleranceProfile の境界値直前／直後をテストする。
- NaN、Infinity、範囲外座標は solver 到達前に拒否する。

### GEO-002 planarization

入力線分を平面化した後、次をすべて満たす。

- 交差点と T-junction が vertex として分割済み
- 許容誤差外の distinct vertex を誤統合しない
- zero-length、重複 edge がない
- 各 crease edge の assignment／role に競合がない
- half-edge の twin／next／face が整合
- 有界面が CCW、面積が正
- Euler 特性が fixture の期待値と一致

### GEO-003 triangulation

- 三角形面積合計と元面積の差が profile 内
- 反転、重複、zero-area triangle、未解消 T-junction がない
- 元 crease が mesh edge として保持される

### TREE-001 tree validity and embedding

- connected、acyclic、葉2以上、正のlength、非負widthを構造妥当性として検査する。葉21以上は構造不正ではなくSupportProfileによる`unsupported`とする。
- node の `rotation` が incident edge の完全な順列である。
- editor position の巡回順変更がrotationとrevision vectorの`design`を更新する。順序が変わらない移動は更新しない。
- label 変更は revision を更新しない。

### TREE-002 symmetry

- mirrored node／edge が一対一で、length、width、topology が同期する。
- axis 上要素が自己対称である。
- OFF 後も両側が残る。
- enable、edit、delete、undo、redo の全操作で dangling `mirrorOf` がない。

### SOLVE-001 result contract

SupportedInputについて、ユーザーキャンセルがなければ次のどちらかだけを返す。

- `verified`：`VerifiedSolution`の証跡再検証が成功
- `no-solution-certified`：方式別certificateの独立checkerが成功

探索打切り、局所最適化失敗、worker crashを`no-solution-certified`に変換しない。葉21、profile数値境界外、非対応FOLDはsolver実行前に`unsupported`とする。

### SOLVE-002 grid approximation

- GEN-BP-GRID-4で`Nx=4`または`Ny=4`を受け付ける。
- columns／rows を別管理し、セルが正方形である。
- `v > 0`のlength/widthは`abs(q-v)/v <= 0.01`、元widthが0なら格子値も厳密に0の候補だけを合格として提案する。
- paper ratioと正方形セルpitchは別のToleranceProfile項目で検査し、枝長/幅の1%へ混同しない。元値、格子値、相対誤差がexpectationと一致する。
- 上限内に候補がない場合は、誤差を隠さず正しい分類を返す。

### SOLVE-003 true box pleating

- fixture が必要とする axial、axial+N、ridge、hinge、gusset を生成する。
- 単なる tree-method CP の座標量子化でないことを role と topology で確認する。
- stable crease ID と source tree edge が全折り線で追跡可能である。

### SOLVE-004 tree method

- active constraint の残差が profile 内である。
- embedded subtree、marker、polygon の対応が完全である。
- 同時イベント、共線イベントの順序が seed や実行環境で変わらない。

## 6. 折り保証 AC

### VERIFY-001 local checks

- 前川、川崎を既知の正例・負例で分類する。
- M/V 変更で前川結果が変わり得る一方、川崎結果は変わらない。
- 表示文言は「局所必要条件」であり「折り可能」と同義にしない。

### VERIFY-002 kinematics

PathCertificateの全segmentで次を満たす。

- 面内距離が`rigidDistanceAbs/Rel`以内で保存される。
- 共有creaseのhinge一致が`hingeCoincidenceAbs`以内、閉路残差が`loopClosureAbs/Angle`以内。
- segment境界で位置とcrease angleが`pathContinuityAbs/Angle`以内で連続し、`timeCoverage`にgapがない。
- `t=0` は平面初期状態。
- 設計projectの`t=1`はtree topology、length ratio、widthを`treeLengthRel/treeWidthRel`以内で満たすuniaxial goal。FOLD projectはresolved assignmentと積層が整合する完全平坦goalを満たす。

### VERIFY-003 continuous collision

- 非隣接 triangle の全 segment に false negative がない。
- NEG-MIDSTEP-CROSS を確実に検出する。
- 許可された共有 hinge 接触を penetration と誤判定しない。
- 接触許容距離を超える penetration を verified にしない。
- broad phase の候補除外が narrow phase の結果を変えないことを小規模全組合せ比較で確認する。

### VERIFY-004 layer order

- 接触・重なりのある face に矛盾のない順序が付く。
- overlap regionごとの`LayerOrderEvent`が接触区間を被覆し、接触中に上下が反転しない。
- cycle のある layer relation を verified にしない。
- CP の表裏 orientation と M/V が path の終状態と整合する。

### VERIFY-005 certificate integrity

- input、mesh、target、path、toleranceのいずれか1 bitを変えるとcertificate hashが一致しない。
- 改変後にhashを攻撃者が再計算したmutationも、reference verifierが拘束/CCD/積層/目標の意味的不整合として拒否する。
- verifier version または ToleranceProfile 変更で stale になる。
- stale／unverified FOLD の metadata を書き換えても、証跡なしに verified として読まない。

### VERIFY-006 manual edit

- M/V edit 直後に verification が stale になる。
- 再検証成功まで verified badge を出さない。
- 再生成時、generationKey 不一致の override を適用せず破棄件数を表示する。

## 7. Worker・状態 AC

### STATE-001 revision race

計算中に入力を編集し、旧jobが後で完了しても、job ID、必要なrevision vector成分、意味的入力hash、profile版のいずれかが不一致の結果を適用しない。

### STATE-002 cancellation

- cancel から規定 UI 応答時間内に worker を terminate する。
- partial result で最後の verified result を上書きしない。
- cancel 後に同じ入力で新規 job を開始できる。

### STATE-003 stale dependencies

- topology、length、width、paper、grid、symmetry、layout、M/Vは依存規則どおり成果物をstaleにする。M/V編集後のCP自体はcurrent、VerificationRecordはstale、集約UIは`unverified`とする。
- name、label、display color は stale にしない。
- save／reload 後も同じ判定になる。

### STATE-004 history

- 1 drag が1 entry。
- solver result 適用が1 entry。
- undo／redo後にrevision vector、derived reference、dirtyが整合する。
- 100件既定、500件上限、byte budget eviction が参照中 blob を壊さない。

### STATE-005 method switch

- method変更では`design`を進め、treeだけを維持し、layout/CP/MV/target/path/layer/verificationを現在状態から破棄して`none`にする。
- 旧成果物をstale表示せず、Undoした場合だけ履歴参照から復元する。

## 8. 保存・復旧 AC

### PERSIST-001 project round-trip

- DesignProject と FoldProject を save/load し、canonical comparison で一致する。
- verified projectのTargetState、PathCertificate、CCD/layer evidence manifestの全hashが同梱EvidenceBlobへ解決し、欠落blobをruntime schemaが拒否する。
- formatVersion 1 fixture を version 2 へ migration する。
- 未知の将来 version は上書きせず read error を表示する。

### PERSIST-002 autosnapshot

- 最終編集から2秒後に snapshot が作成される。
- 途中書込みを模擬しても直前の完全 snapshot を復元できる。
- current pointerの原子的切替を検査する。最近一覧は`updatedAt`降順10件とし、11件目はopen/lock中を保護して最古から決定的にevictする。
- 起動時は自動適用せず復元／破棄を選ばせる。

### PERSIST-003 multi-tab

- 同一 project の2タブ目が read-only になる。
- takeover 後に旧タブが編集を継続できない。
- heartbeat消失時も確認なしに編集開始しない。競合snapshotを更新日時/検証状態付きで比較し、どちらかを明示選択でき、自動mergeしない。

### PERSIST-004 save fallback

- 上書き API 使用時は同一 file へ保存する。
- fallbackはdownloadを開始して確認を表示し、開始だけではdirtyを解除しない。
- cancel/permission denial/「未保存のまま」でdirtyを解除せず、上書きAPI成功または利用者の「ダウンロードを確認しました」後だけ解除する。

### PERSIST-005 PWA update

- dirty 状態で service worker が自動 reload しない。
- 更新通知から保存後に reload できる。
- migration 失敗時に旧 snapshot を保持する。

### PERSIST-006 PWA install capability

- ブラウザがinstall prompt/APIを提供する場合はインストール導線が動く。
- 提供しない対応ブラウザでは公開URL利用を正式経路として案内し、主要機能を制限しない。

## 9. FOLD・export AC

### FOLD-001 import subset

- FOLD 1.1／1.2のtop-level key frameで、`file_frames`が不存在または空のfixtureを読める。内部では正規化済みFOLD 1.2とsource spec/hashを保持する。
- faces なし fixture から面を再構築できる。
- 1件以上の`file_frames`、3D、non-planar、non-manifold、C/Jを理由付きで拒否する。
- runtime schema が duplicate／out-of-range index／非有限値を拒否する。

### FOLD-002 parallel arrays

`edges_vertices`、assignment、custom ID、role、source arraysの長さが一致し、全face indexが範囲内である。未知custom fieldはトポロジ不変round-tripでは保持し、再平面化で対応が壊れる場合は警告して除外する。

### FOLD-003 external compatibility

- 出力 FOLD 1.2 を少なくとも2つの独立した対応 viewer／tool で開ける。
- 互換確認 tool と version を fixture README に固定する。
- custom field を無視しても標準 CP として読める。

### FOLD-004 imported verification

- `.fold`読込直後はcustom verification metadataに関係なく`unverified`である。
- REF-FOLD-FLAT-VERIFIABLEはFOLD検証用SupportProfileを通り、flat target、PathCertificate、CCD、動的/終端積層、VerificationRecordを生成してreference verifierに合格する。
- FOLD profile外は理由付き`unsupported`とするが、表示、M/V編集、警告付き再出力、平面/未検証候補3Dを許可する。

### EXPORT-001 SVG

- viewBox、境界、M/V/B/F、線種、role が期待値と一致する。
- label の特殊文字が markup として実行されない。

### EXPORT-002 PNG

- 既定長辺4,096px、最大16,384px。
- 白／透明背景を選べる。
- aspect ratio、線切れ、余白外描画がない。

### EXPORT-003 PDF

- A4／A3／Letter、余白10mm、1 page、auto orientation。
- 余白内へ最大 fit し、crop しない。
- PDF 内の線種と色が SVG と意味的に一致する。

### EXPORT-004 unverified warning

- unverified/staleは明示同意なしに出力しない。
- FOLD/SVGは状態、revision vector、profile情報をmetadataへ持ち、PNG/PDFは可視footerを持つ。
- export metadataの`verified`文字列だけを改変して再importしてもverifiedにならない。

## 10. UI・アクセシビリティ AC

### UI-001 verification states

none、stale、unverified、verifying、verified、no-solution-certified、unsupported、errorを色だけでなくtext/iconで識別できる。cancelledは永続状態にせず、直前状態を維持した通知として識別できる。

### UI-002 path playback

- slider は path time `t` を表す。
- crease ごとに異なる角度で動く fixture を正しく再生する。
- play、pause、reverse、seek 後も certificate path と同じ状態になる。

### UI-003 keyboard

- keyboard だけで node／edge／crease の選択、追加、編集、削除、tab 移動ができる。
- editor focus 中に global shortcut が誤発火しない。
- undo、redo、save shortcut が Windows で動く。

### UI-004 responsive boundary

- 1280px 以上で3 pane が重ならない。
- 1280px 未満は明確な利用警告を出す。
- browser zoom 200% で主要 form と状態文言へ到達できる。

### A11Y-001

- 通常 UI／form で automated WCAG 2.2 AA violation がない。
- mountain／valley／error／active を color 以外でも区別する。
- 3D の既知制限を text で提示し、検証値を DOM から読める。

## 11. ネットワーク・ライセンス AC

### NFR-NET-001

E2E 中の network request を記録し、同一 origin の静的 asset 以外が0件である。analytics、広告、外部 font、CDN、crash upload を含めない。

### NFR-LIC-001

- production/development dependencyのSPDX identifierが`MIT`、`0BSD`、`BSD-2-Clause`、`BSD-3-Clause`、`Apache-2.0`、`ISC`のallowlist内にある。
- GPL／AGPL／LGPL の直接・推移依存を CI で拒否する。
- ISC（earcut 等）を許可する。
- fixture と参考実装の由来・license を inventory 化する。

### NFR-SEC-001

- CSP が inline script と未許可接続先を拒否する。
- project name／label／imported metadata の XSS fixture が無害化される。
- oversized input が solver／renderer へ到達する前に拒否される。

### NFR-LIMIT-001

`runtime-limits-v1.json`のfile bytes、頂点/辺/面/文字列、Nx/Ny、履歴byte budget、入力桁数/最小特徴量/最大比率の境界値と直前/直後を検査し、超過時にデータを壊さず理由を表示する。

## 12. 性能 AC

基準端末は Windows 11、4 core CPU、16GB RAM、integrated GPU、1920×1080 とし、M0F で具体 CPU 型番と browser version を固定する。

### PERF-001 candidate targets

- 箱ひだ配置3秒、ツリー法配置30秒は努力目標として計測する。
- 超過しても solver を強制 timeout せず、progress と cancel が機能する。
- 葉4／8／20 fixture ごとの中央値、p95、peak memory を記録する。

### PERF-002 verification

- hard timeout を設けない。
- phase ごとの時間、探索量、CCD query 数、peak memory を出力する。
- 同一 fixture／version の回帰を CI trend で検出する。

### PERF-003 3D

- 必須：1,000 triangles、基準端末、viewport 1920×1080 で30fps以上。
- 努力目標：2,000 triangles で30fps以上。
- 5秒 warm-up 後10秒を測定し、average と1% low を記録する。

### QUALITY-001 coverage

`model/`、`geometry/`、`solver/`、`planner/`、`verify/`、`referenceVerifier/`のunit test coverageが80%以上で、property/mutation testを別に集計する。

### QUALITY-002 memory

固定長時間シナリオでWorker、3D、履歴blob、snapshotのretained memoryを測定し、M0Fで固定する回帰上限以内である。
