# Single-hinge rotation swept-AABB step candidate v1

Status: topology- and pose-bound conservative broad phase for one declared
single-hinge step on one dyadic slab. It is not an angle schedule, nonlinear
narrow-phase CCD, a legal-contact rule, a self-intersection decision, a
collision-free result, a product `verified` result, or evidence for global M0F
`GO`.

## Closed input contract

`computeSingleHingeRotationSweptAabbStepV1` accepts exactly two root fields:

- `bindingInput`, the complete input to
  `bindStaticRationalTrianglePoseToFoldMeshV1`; and
- `step`, containing bounded stable path, step, active-edge, and moving-side
  IDs plus one canonical dyadic slab.

Unknown fields, accessors, non-plain objects, revoked proxies, invalid IDs, and
malformed slabs fail atomically. Binding failure is retained as a binding-stage
diagnostic. The function never reads a caller getter and returns an owned,
deeply frozen result.

The mesh binding first establishes a bijection between every reconstructed
FOLD triangle and one exact-rational 3D pose triangle. It also checks a single
exact point for each mesh vertex, exact source-face metric rigidity and
coplanarity, mesh edge/face incidence, declared hinge segments, stable triangle
order, and complete unordered-pair order. Caller revision IDs remain labels;
they are not cryptographic source provenance.

## Supported topology slice

The active edge must be one two-face declared hinge. Exactly one declared hinge
segment may join that source-face pair in v1. The requested moving-side face
must be incident to the edge.

The implementation builds the source-face dual graph from all other declared
hinges, removes the active face-pair adjacency, and traverses from the requested
moving-side face. Success requires the opposite incident face to be unreachable.
Consequently the active hinge is a bridge and the derived moving and stationary
components are both nonempty. A hinge on a dual cycle and a multi-segment hinge
fail closed; the caller cannot supply or override component membership.

This bridge restriction is deliberately narrower than general rigid origami.
It does not cover simultaneous hinges, closed kinematic loops, or composed
motion generated by earlier steps.

## Conservative primitive construction

Every bound mesh triangle becomes exactly one motion primitive.

For a triangle on the moving component, the first exact endpoint of the active
hinge segment is the common origin `q0 = q1`. Each local vertex is computed by
exact projective-rational subtraction from that point. The inherited motion
family encloses every `SO(3)` rotation about the point. A rotation about the
declared fixed hinge axis is a subset of that enclosure, but no axis-angle
function or fold direction is constructed or checked.

For a stationary triangle, its first exact vertex is used as `q0 = q1` and all
three exact local offsets are retained. The inherited arbitrary-`SO(3)` bound
about that vertex contains the required identity pose. It is intentionally
looser than an exact stationary AABB, but substantially tighter than rotating
the same world coordinates about the world origin.

The census treats primitive rotations independently. Real triangles on one
rigid component share a rotation, so this independence only enlarges the
enclosure; it cannot justify promoting an overlap candidate to separation.

## Complete bounded broad-phase ledger

The existing affine-origin rotation census sorts the complete bound triangle
set and records every unordered pair. The step then joins each row to the mesh
pair ledger by both triangle IDs and `pairIndex`. Any missing or reordered row
fails the whole step.

A strict exact gap on one swept-AABB axis remains
`separated-by-certified-swept-aabb`. Box overlap and arithmetic
indeterminacy remain unresolved. A non-same-face row requires later narrow
phase unless it has a strict separation certificate. Same-source-face rows are
kept in the complete ledger but are not promoted to inter-face narrow-phase
candidates by this component.

The inherited ceiling is 64 triangles and 2,016 unordered pairs. Complete
enumeration means only complete coverage of this accepted bound pose and this
single supplied slab. The result does not prove that slabs cover an entire
path.
