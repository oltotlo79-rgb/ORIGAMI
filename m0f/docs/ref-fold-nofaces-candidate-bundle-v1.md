# REF-FOLD-NOFACES project-authored candidate bundle v1

Status: reproducible project-authored candidate vector for the exact face
reconstruction, separate face-complex audit, persisted audit-evidence reaudit,
and eleven-case semantic mutation stages. It is not a canonical fixture or a
scientific verification record.

## Location and catalog isolation

The bundle is stored under
`tests/candidate-vectors/REF-FOLD-NOFACES/`, outside the canonical
`tests/fixtures/manifest.json` repository. The canonical manifest is not
modified and contains no `REF-FOLD-NOFACES` entry. The verifier checks that
absence but does not run the canonical repository gate or the global M0F gate.

The source is a project-authored FOLD 1.2 crease-pattern document with six
vertices, six boundary edges, and no `faces_vertices`. It represents only the
closed NOFACES adapter boundary used by the candidate reconstruction work.

## Closed ledger and provenance

`candidate-bundle-ledger.json` is governed by the closed runtime parser in
`ref-fold-nofaces-candidate-bundle.ts` and the matching Draft 2020-12 schema
`ref-fold-nofaces-candidate-bundle-ledger-v1.schema.json`. Both contracts fix:

- `contractStatus: "candidate"` and `scientificClaim: false`;
- canonical-manifest registration to `not-registered`;
- canonical promotion, ToleranceProfile inclusion, and scientific verification
  claims to `false`;
- `globalM0fGate: "not-evaluated"`;
- the exact ordered set of eight artifact IDs, roles, paths, media types, and
  MIT license declarations;
- project-authored or deterministic-generated provenance and the ordered
  upstream artifact dependencies for each row; and
- the SHA-256 digest of every payload artifact's committed UTF-8 bytes.

The ledger itself is schema- and parser-validated and must be byte-identical to
deterministic regeneration. It is not included in its own hash list, avoiding
a self-referential digest. Unknown fields such as `verified`, an invented
`toleranceProfile`, or a GO claim are rejected. Claim escalation and any
artifact identity, order, path, provenance, dependency, or hash rewrite are
also rejected.

## Payload artifacts

The eight hash-bound payloads are:

1. `input.fold`, the project-authored source;
2. the bundle-local `README.md` claim boundary;
3. the typed candidate evidence index;
4. the exact reconstruction experiment record;
5. the separately executed face-complex audit experiment record;
6. independent-auditor evidence with its current source-set hash;
7. the canonical eleven-case semantic mutation suite; and
8. the saved deterministic mutation result.

The evidence index, experiment runner, FOLD adapter, reconstruction engine,
audit-input adapter, independent auditor evidence parser/reauditor, and mutation
suite parser/runner are reused directly. The bundle runner does not duplicate
their geometry or audit logic.

## Replay and deterministic regeneration

Read-only verification is:

```text
npx tsx m0f/ref-fold-nofaces-candidate-bundle-cli.ts --verify
```

Run these commands from the repository root. The verifier intentionally checks
the repository-relative canonical manifest to ensure this candidate ID remains
unregistered.

Deterministic regeneration followed by immediate verification is:

```text
npx tsx m0f/ref-fold-nofaces-candidate-bundle-cli.ts --write
```

The generator uses stable JSON with a final LF, reconstruction seed 41 and
repetition 0, and audit seed 42 and repetition 0. It regenerates the source,
README, all evidence, and the ledger. A directory containing an unexpected
file, directory, or link is refused rather than cleaned.

Verification requires the exact nine-file directory, checks all eight ledger
hashes, parses every saved contract, adapts the committed source, reruns exact
reconstruction and the separate audit, reaudits persisted audit evidence
against the current independent-auditor source-set hash, reruns all eleven
mutations in canonical order, and compares every generated byte including the
ledger. A coherently rehashed engine/seed or saved-result rewrite still differs
from deterministic regeneration and is blocked.

This file consumer is intentionally scoped to a local committed candidate
directory. It rejects linked entries and unexpected directory contents, but it
does not claim the file-handle identity and concurrent-replacement defenses of
the manifest-bound consumer. It must not be used as an untrusted, concurrently
mutable artifact service.

## Claim boundary

A reproducible result means only that this fixed source reproduces its saved
candidate reconstruction, audit, independent audit evidence, and mutation
regression outputs with the current code. It does not establish general FOLD
compatibility, foldability, a rigid folding path, contact legality, collision
freedom, layer ordering, target attainment, or product readiness.

No ToleranceProfile exists in this bundle. No canonical promotion or scientific
verification is claimed. The global M0F gate remains explicitly not evaluated.
